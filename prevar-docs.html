x4c's architecture

memory size

each word is 32b

the first 4 bits are the instruction opcode

in the ic7m format:

the next 2 bit reference the "buffer" or cpu register used in the operation

and the last 26 bits reference the address

icmmmmmmm

a3002a1b0


the other format used is i3c format in this format:

the next 6 bits split into three parts are used to reference three cpu buffers.

iccc

a120


the total size of memory is 8MiB or 2,097,152 words


buffers

there are 4 cpu buffer's(registers) accessable. all buffers(0,1,2,3) are general purpose however buffer 3 will contain a callback address whenever a jump command is used

instructions

i = instruction

c = cpu buffer

m = memory address

0 noop

increments program counter

1 jumpu

sets buffer[3] to current address, jumps to m

2 jumpc

sets buffer[3] to current address, m jumps to m if c = 0xffffffff

3 jumpp

sets buffer[3] to current address, jumps to c

the following all increment program counter after the instruction

4 take

sets the register at c to the value at the address of m

5 place

interpets m as a pointer to pointer and sets that location to c equvalent to ( [[m]] = c or **m = c )

6 gthan

if c[0] is greater than c[1] then c[2] will be set to 0xffffffff, otherwise it will be set to 0x0

7 lthan

if c[0] is less than c[1] then c[2] will be set to 0xffffffff, otherwise it will be set to 0x0

8 and

performs a bitwise and on c[0] and c[1] then stores the result in c[2]

9 or

performs a bitwise or on c[0] and c[1] then stores the result in c[2]

10 xor

performs a bitwise xor on c[0] and c[1] then stores the result in c[2]

11 nor

performs a bitwise nor on c[0] and c[1] then stores the result in c[2]

12 add

performs an addition operation on c[0] and c[1] then stores the result in c[2]

13 sub

performs a subtraction operation on c[0] and c[1] then stores the result in c[2]

14 shiftl

shifts c[0] left by c[1] voiding overflow and storing the result in c[2]

15 shiftr

shifts c[0] right by c[1] voiding overflow and storing the result in c[2]

vram offset and size

the vram begins at 0x03ffc000(67092480) and continues for 0x4000(128*128) pixels ending at 0x04000000(67108864)

input offset

input byte is set at 0x003ffbbb and will be updated when a new to character is pressed

Syntax

Goal of the syntax

to minimise the use of symbolic characters in favour of words

Comments

# this is a comment

Statements

statements come in two formats ic7m and i3c

ic7m format:
take 2 124
i3c format:
1 and 3 2

in the current build(prevar) only integers are supported

Commands

regardless of architecture all commands immitate x4c's capability so available keywords are

ic7m:
noop, jumpu, jumpc, jumpp, take, place
i3c:
gthan, lthan, and, or, xor, nor, add, sub, shiftl, shiftr
